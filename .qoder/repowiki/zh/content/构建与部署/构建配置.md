## 项目构建配置概述

本项目采用 Vite 作为核心构建工具，通过 `vite.config.ts` 文件进行基础的构建配置。配置文件定义了开发服务器、React插件支持和模块解析等关键功能。结合 `package.json` 中的脚本命令，形成了完整的开发与构建工作流。当前配置架构相对简化，专注于提供高效的开发体验和基本的构建功能。

## 路径别名与模块解析

`resolve.alias` 配置定义了模块导入的路径别名，其中 `@` 被映射到 `src` 目录的绝对路径。这一配置极大地简化了模块导入语句，提升了代码的可读性和可维护性。在项目中，开发者可以使用 `@/components/Button` 这样的简洁路径来导入位于 `src/components/Button` 的组件。

需要注意的是，TypeScript 也在 `tsconfig.json` 中配置了类似的路径映射（`"@/*": ["src/*"]`），确保了开发工具和构建工具的一致性。这种双重配置确保了在开发环境中无论是通过 Vite 还是 TypeScript 编译器，都能正确解析路径别名。

```mermaid
flowchart LR
A["源代码: import Layout from '@/layout/BasicLayout'"] --> B["Vite 解析"]
B --> C["resolve.alias: '@' -> './src'"]
C --> D["TypeScript 路径映射"]
D --> E["实际路径: ./src/layout/BasicLayout"]
E --> F["成功加载模块"]
```

## CSS 预处理器配置

项目使用 Tailwind CSS 作为 CSS 框架，并通过 `src/styles/tailwind.css` 文件进行配置。该文件简单地导入了 tailwindcss，实现了基础的样式框架功能。相比之前的 Less 配置，当前版本采用了更简洁的样式处理方式。

Tailwind CSS 提供了实用优先的 CSS 框架，通过原子化的类名来快速构建用户界面，减少了手写 CSS 的需求。配合 Vite 的快速构建能力，为开发者提供了高效的样式开发体验。

## 构建脚本与工作流

`package.json` 文件中的 `scripts` 字段定义了项目的标准化工作流。`start` 脚本通过 `vite` 命令启动开发服务器，利用 Vite 的快速冷启动和热更新特性。`build` 脚本执行 `vite build`，触发生产环境的构建流程。`serve` 脚本则通过 `vite preview` 启动一个本地服务器来预览已构建的产物。

这些脚本与 `vite.config.ts` 中的配置项协作：开发服务器的端口由 `server.port` 指定；插件配置决定了开发和构建时的代码转换行为；路径别名和 CSS 配置确保了开发时的正确解析。整个工作流形成了一个从开发到部署的完整闭环，但相比之前的复杂配置，当前版本更加简洁高效。
