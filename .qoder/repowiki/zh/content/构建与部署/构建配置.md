## TypeScript 配置更新

项目已升级到 TypeScript 6，采用了现代化的配置策略。主要更新包括：

### 模块解析策略
- **moduleResolution**: 更新为 `"bundler"`，提供更好的打包性能和兼容性
- **移除 baseUrl**: 简化路径解析，减少配置复杂度
- **增强的模块系统**: 支持现代 ES 模块和 CommonJS 的混合使用

### 路径映射配置
TypeScript 配置了全面的路径映射，支持多种模块结构：
- `@/*`: 根目录映射到 `./src/*`
- `@/components/*`: 组件目录映射
- `@/pages/*`: 页面目录映射  
- `@/services/*`: 服务层映射
- `@/store/*`: 状态管理映射
- `@/utils/*`: 工具函数映射
- `@/types/*`: 类型定义映射

### TypeScript 6 特性
- **ESNext 模块**: 使用最新的 ECMAScript 模块语法
- **严格类型检查**: 启用了全面的类型安全检查
- **实验性装饰器**: 支持装饰器语法
- **增量编译**: 提升编译性能

## 路径别名与模块解析

`resolve.alias` 配置定义了模块导入的路径别名，其中 `@` 被映射到 `src` 目录的绝对路径。这一配置极大地简化了模块导入语句，提升了代码的可读性和可维护性。

### Vite 路径映射
```typescript
resolve: {
  alias: {
    "@": path.resolve(__dirname, "src"),
  },
},
```

### TypeScript 路径映射
TypeScript 在 `tsconfig.json` 中配置了更详细的路径映射：
```json
"paths": {
  "@/*": ["./src/*"],
  "@/components/*": ["./src/components/*"],
  "@/pages/*": ["./src/pages/*"],
  "@/services/*": ["./src/services/*"],
  "@/store/*": ["./src/store/*"],
  "@/utils/*": ["./src/utils/*"],
  "@/types/*": ["./src/types/*"]
}
```

### Jest 路径映射
Jest 配置中也包含了相应的路径映射：
```typescript
moduleNameMapper: {
  "@/(.*)": "<rootDir>/src/$1",
}
```

这种多层配置确保了在开发、构建、测试等不同环境中都能正确解析路径别名。

## 构建脚本与工作流

`package.json` 文件中的 `scripts` 字段定义了项目的标准化工作流。`start` 脚本通过 `vite` 命令启动开发服务器，利用 Vite 的快速冷启动和热更新特性。`build` 脚本执行 `vite build`，触发生产环境的构建流程。`serve` 脚本则通过 `vite preview` 启动一个本地服务器来预览已构建的产物。

### 开发环境配置
- **端口**: 3123
- **热更新**: Fast Refresh
- **TypeScript 支持**: 即时类型检查

### 测试环境配置
项目同时支持 Jest 和 Vitest 测试框架：
- **Jest**: 用于组件测试和集成测试
- **Vitest**: 用于单元测试和组件测试
- **测试环境**: jsdom，模拟浏览器环境

### 构建优化
- **模块解析**: 使用 bundler 模式提升性能
- **路径映射**: 多层配置确保一致性
- **TypeScript 6**: 利用最新语言特性和性能优化
