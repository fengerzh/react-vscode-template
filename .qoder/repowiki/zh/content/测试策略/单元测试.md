## 简介
本项目采用 Vitest 作为主要的单元测试框架，结合 React Testing Library、Ant Design 的测试工具和 Axios Mock Adapter 等技术，构建了完整的前端测试体系。测试覆盖了 React 组件、布局组件、页面组件、路由系统、服务层 API 调用和 Zustand 状态管理等核心功能模块。通过合理的测试配置和模拟策略，确保了代码质量和功能稳定性。

**更新** 项目已从 Jest 完全迁移到 Vitest，提供了更快的测试执行速度和更好的 TypeScript 支持。新增的测试覆盖范围包括布局组件、页面组件和路由系统的完整测试，进一步完善了测试体系。

### Vitest配置详解

**Vitest配置特点**：
- 使用 jsdom 作为测试环境，模拟浏览器 DOM 环境
- 通过 setupFiles 引入 jest.setup.ts 进行全局测试环境初始化
- 支持 TypeScript 和 JSX 文件的自动检测和编译
- 配置了 v8 覆盖率提供者，支持多种覆盖率报告格式
- 通过别名映射简化模块导入路径

**测试文件组织**：
- 自动包含 src 目录下所有 .test.ts、.test.tsx、.spec.ts、.spec.tsx 文件
- 排除 Cypress 端到端测试目录和用户布局的 Cypress 测试文件
- 支持模块别名 @ 指向 src 目录

**配置优化**：
- 移除了 Jest 特定的 transform 配置，因为 Vitest 内置支持 TypeScript
- 简化了 moduleNameMapper 配置，直接使用 Vitest 的别名系统
- 保留了必要的 setupFiles 配置以兼容现有的测试初始化代码
- 新增了对 Cypress 测试文件的排除规则
